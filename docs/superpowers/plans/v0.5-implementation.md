# kata-agent v0.5 Providers, API Automation, and Static Scan Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add explicit Claude and OpenAI provider adapters, a full API automation workflow, and read-only static scan artifacts that can feed implementation-aware test design.

**Architecture:** Keep the Workflow Engine as the only flow controller. Providers stay behind `ProviderAdapter` and are selected by runtime configuration; API automation ships as a separate `api-script-gen` workflow so existing web automation remains web-only; static scan is a read-only source-repo plugin that produces schema-backed risk artifacts and optional `TestSpecAuthorInput` context. All external effects are manifest-declared, schema-validated, and exercised through deterministic tests with mocked network or temp repositories.

**Tech Stack:** TypeScript, Bun workspaces, Bun test, Ajv JSON Schema validation, YAML manifests, existing Artifact Repository / Workflow Engine / Plugin Runtime contracts, Fetch API, OpenAI Chat Completions-compatible HTTP, Anthropic Messages-compatible HTTP, read-only Git diff scanning.

## Scope Source

This plan implements the requested v0.5 slice:

- Claude + OpenAI providers.
- API automation.
- Static scan.

It also resolves the relevant extension points from `docs/superpowers/specs/2026-05-01-kata-agent-architecture-design.md` §22:

- Claude / OpenAI provider adapters.
- API automation as a full surface.
- Static-scan risk injection into test-case generation.

Included:

- Add first-class `OpenAIProvider` and `ClaudeProvider` adapters while keeping `OpenAICompatibleProvider` for custom OpenAI-shaped endpoints.
- Add provider selection through CLI/runtime config without changing agent prompts or workflow flow control.
- Add schema-backed API automation contracts: `ApiScriptGenInput`, `ApiFlowSpec`, `ApiRunPlan`, and `ApiRunRecord`.
- Add an `http-api` automation plugin with mock and real `fetch` handlers.
- Add an `api-script-gen` skill, workflow, CLI command, builders, gates, evidence collection, and report output.
- Add schema-backed static scan contracts: `SourceRepoRef`, `StaticScanInput`, `RiskPoint`, and `InspectionReport`.
- Add a read-only `static-scan` plugin that scans explicit diffs/source refs and writes `InspectionReport`.
- Add optional `inspectionReportRef` to `TestSpecAuthorInput` so risk points can guide test-case generation only when explicitly provided.
- Add tests proving provider secrets do not leak, API automation does not weaken web-only constraints, and static scan never mutates source repositories.

Excluded:

- Codex and Hermes provider adapters.
- Provider streaming, tool calling, prompt caching implementation, or cost accounting beyond current usage metadata.
- Mobile or desktop automation.
- Browser UI automation changes outside the existing `ui-script-gen` path.
- Source code mutation, auto-hotfixes, autonomous PR creation, or write access to source repositories.
- Static scan rules that require external services, package manager audits, or network access.
- Automatic conversion of static scan findings into issues; Zentao sync remains driven by explicit `IssueDraft` artifacts.

## Hard Constraints

- Keep `WorkflowExecutor` as the only workflow flow controller.
- Keep workflow node types exactly: `tool`, `agent`, `gate`, `human`, `artifact`.
- Do not remove or weaken the existing `OpenAICompatibleProvider`; explicit OpenAI and Claude providers are additive.
- Provider adapters must never include secrets in `ProviderResponse.raw`, thrown error messages, trace events, artifact output, or test snapshots.
- Runtime provider selection must fail with `MISSING_SECRET provider config` when the selected real provider lacks required config.
- `ui-script-gen` must remain web-only. API automation must use a separate `api-script-gen` skill/workflow.
- API automation must never accept mobile or desktop surfaces.
- API automation real mode must use mocked `fetch` in tests and must not call external services.
- Static scan must be read-only. It must not write, format, patch, stage, commit, or reset source repositories.
- `SourceRepoRef.sourceRoot` must be relative to the kata workspace root; schema and runtime must reject absolute paths and `..`.
- Static scan must not persist raw absolute paths, credentials, bearer tokens, cookies, or internal service URLs.
- Static scan findings must be explicit `RiskPoint` objects inside `InspectionReport`; they must not directly create `IssueDraft` artifacts.
- Plugin manifests must reference schemas from `SCHEMA_REGISTRY`.
- Plugin actions must declare side effects and required secrets in `plugin.yaml`.
- File tests must use temp dirs and clean them up.

```ts
import type { JsonValue } from "../../core/src/index";
import type { ProviderAdapter, ProviderRequest, ProviderResponse } from "./provider";

export interface ClaudeProviderOptions {
  apiKey: string;
  model: string;
  baseUrl?: string;
  fetchImpl?: typeof fetch;
}

export class ClaudeProvider implements ProviderAdapter {
  readonly id = "claude";
  readonly capabilities = {
    toolUse: false,
    structuredOutput: true,
    promptCaching: false,
    streaming: false,
    maxContextTokens: 200000,
  };

  private readonly baseUrl: string;
  private readonly fetchImpl: typeof fetch;

  constructor(private readonly options: ClaudeProviderOptions) {
    this.baseUrl = (options.baseUrl ?? "https://api.anthropic.com/v1").replace(/\/+$/, "");
    this.fetchImpl = options.fetchImpl ?? fetch;
  }

  async generate(request: ProviderRequest): Promise<ProviderResponse> {
    const started = Date.now();
    const system = request.messages
      .filter((message) => message.role === "system")
      .map((message) => message.content)
      .join("\n\n");
    const messages = request.messages
      .filter((message) => message.role !== "system")
      .map((message) => ({ role: message.role, content: message.content }));

    const response = await this.fetchImpl(`${this.baseUrl}/messages`, {
      method: "POST",
      headers: {
        "content-type": "application/json",
        "x-api-key": this.options.apiKey,
        "anthropic-version": "2023-06-01",
      },
      body: JSON.stringify({
        model: this.options.model,
        max_tokens: request.maxTokens ?? 4096,
        temperature: request.temperature ?? 0,
        system: system || undefined,
        messages,
        stop_sequences: request.stopSequences,
      }),
    });
    if (!response.ok) throw providerError(response.status);

    const json = (await response.json()) as JsonValue & {
      content?: Array<{ type?: string; text?: string }>;
      usage?: { input_tokens?: number; output_tokens?: number };
    };
    const content = json.content?.find((item) => item.type === "text")?.text;
    if (typeof content !== "string") throw new Error("INVALID_MODEL_JSON missing content");
    return {
      content,
      usage: {
        inputTokens: json.usage?.input_tokens ?? 0,
        outputTokens: json.usage?.output_tokens ?? 0,
        durationMs: Date.now() - started,
      },
      raw: json,
    };
  }
}

function providerError(status: number): Error {
  if (status === 401 || status === 403) {
    return new Error(`MISSING_SECRET provider authentication failed: ${status}`);
  }
  if (status >= 400 && status < 500 && status !== 429) {
    return new Error(`SCHEMA_VALIDATION_FAILED provider request failed: ${status}`);
  }
  return new Error(`PROVIDER_TRANSIENT ${status}`);
}
```

Create `packages/agent-runner/src/provider-factory.ts`:

```ts
import type { LocalConfigLoader } from "../../core/src/index";
import { ClaudeProvider } from "./claude-provider";
import { MockProvider } from "./mock-provider";
import { OpenAICompatibleProvider } from "./openai-compatible-provider";
import { OpenAIProvider } from "./openai-provider";
import type { ProviderAdapter } from "./provider";

export type ProviderMode = "mock" | "openai-compatible" | "openai" | "claude";

export function createConfiguredProvider(
  mode: ProviderMode,
  config: LocalConfigLoader,
  mockResponses: Record<string, string>,
): ProviderAdapter {
  if (mode === "mock") return new MockProvider(mockResponses);
  if (mode === "openai") {
    const apiKey = config.resolveSecret("OPENAI_API_KEY");
    const model = config.resolveSecret("OPENAI_MODEL");
    if (!apiKey || !model) throw new Error("MISSING_SECRET provider config");
    return new OpenAIProvider({
      apiKey,
      model,
      baseUrl: config.resolveSecret("OPENAI_BASE_URL"),
    });
  }
  if (mode === "claude") {
    const apiKey = config.resolveSecret("ANTHROPIC_API_KEY");
    const model = config.resolveSecret("ANTHROPIC_MODEL");
    if (!apiKey || !model) throw new Error("MISSING_SECRET provider config");
    return new ClaudeProvider({
      apiKey,
      model,
      baseUrl: config.resolveSecret("ANTHROPIC_BASE_URL"),
    });
  }

  const baseUrl = config.resolveSecret("KATA_AGENT_PROVIDER_BASE_URL");
  const apiKey = config.resolveSecret("KATA_AGENT_PROVIDER_API_KEY");
  const model = config.resolveSecret("KATA_AGENT_PROVIDER_MODEL");
  if (!baseUrl || !apiKey || !model) throw new Error("MISSING_SECRET provider config");
  return new OpenAICompatibleProvider({
    id: "openai-compatible",
    baseUrl,
    apiKey,
    model,
  });
}
```

```ts
import type {
  ApiFlowSpec,
  ApiRunPlan,
  ApiRunRecord,
  ArtifactRef,
  EvidenceKind,
  EvidencePack,
  TestSpec,
} from "../../domain/src/index";

function apiFlowId(index: number): string {
  return `API-FLOW-${String(index + 1).padStart(3, "0")}`;
}

function evidenceKind(path: string): EvidenceKind {
  if (path.includes("/request-")) return "api-request";
  if (path.includes("/response-")) return "api-response";
  return "run-log";
}

export function buildApiFlowSpecFromTestSpec(
  testSpecRef: ArtifactRef,
  spec: TestSpec,
): ApiFlowSpec {
  const flows: ApiFlowSpec["flows"] = [];
  for (const module of spec.modules) {
    for (const testCase of module.cases) {
      if (
        testCase.automation.surface !== "api" ||
        testCase.automation.readiness !== "ready"
      ) {
        continue;
      }
      flows.push({
        id: apiFlowId(flows.length),
        title: testCase.title,
        testCaseId: testCase.id,
        priority: testCase.priority,
        surface: "api",
        requests: testCase.steps.map((step) => ({
          id: step.id,
          method: step.action.toUpperCase().startsWith("GET") ? "GET" : "POST",
          path: step.action.includes(" ") ? step.action.split(" ").at(-1)! : `/${spec.feature}`,
          expectedStatus: Number.parseInt(step.expected, 10) || 200,
        })),
        assertions: testCase.assertions.map((assertion) => ({
          id: assertion.id,
          layer: assertion.layer,
          kind: assertion.kind === "header" ? "header" : assertion.kind === "schema" ? "schema" : assertion.kind === "json-body" ? "json-body" : "status",
          target: assertion.target,
          expected: assertion.expected,
          requirementRefs: [...assertion.requirementRefs],
        })),
      });
    }
  }
  return {
    schemaVersion: "0.1",
    project: spec.project,
    feature: spec.feature,
    sourceTestSpecRef: testSpecRef.id,
    flows,
  };
}

    runner: "http-api",
    mode,
    baseUrl,
    sourceApiFlowSpecRef: flowSpecRef.id,
    flows: flow.flows,
  };
}

  runRecordRef: ArtifactRef,
```

```ts
import type { RiskPoint } from "@kata-agent/domain";

interface DiffLine {
  file: string;
  line?: number;
  text: string;
}

const SECRET_PATTERN = /(Bearer\s+[A-Za-z0-9._-]+|api[_-]?key\s*[:=]\s*["'][^"']+["']|token\s*[:=]\s*["'][^"']+["'])/i;
const TEST_WEAKENING_PATTERN = /(test\.skip|describe\.skip|expect\([^)]*\)\.toBeTruthy\(\)|assert\.ok\(true\))/;

export function scanDiffLines(lines: DiffLine[]): RiskPoint[] {
  const risks: RiskPoint[] = [];
  for (const line of lines) {
    if (SECRET_PATTERN.test(line.text)) {
      risks.push({
        id: `RISK-${String(risks.length + 1).padStart(3, "0")}`,
        category: "security",
        severity: "P0",
        file: line.file,
        line: line.line,
        title: "Hardcoded secret-like literal",
        evidence: line.text,
        recommendation: "Read secrets from runtime configuration and keep artifacts redacted.",
        sourceRefs: [`${line.file}${line.line ? `:${line.line}` : ""}`],
        testFocus: ["Verify secret values are absent from reports, traces, and provider/plugin outputs."],
      });
    }
    if (TEST_WEAKENING_PATTERN.test(line.text)) {
      risks.push({
        id: `RISK-${String(risks.length + 1).padStart(3, "0")}`,
        category: "test-weakening",
        severity: "P1",
        file: line.file,
        line: line.line,
        title: "Test assertion appears weakened",
        evidence: line.text,
        recommendation: "Keep executable assertions tied to requirement refs and observable behavior.",
        sourceRefs: [`${line.file}${line.line ? `:${line.line}` : ""}`],
        testFocus: ["Add or restore a concrete assertion for the changed behavior."],
      });
    }
  }
  return risks;
}
```
